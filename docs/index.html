Key Concepts

Workflow Orchestration

  • Coordinates tasks in an orderly sequence
  • Ensures that tasks are executed in the correct order
  • Reacts according to IFTTT conditions
  • It involves both manual and automated steps

Workflow Orchestration

It is a set of sequences of tasks, and data processes between those tasks

Workflow Orchestration

  • Orchestration is configuring, managing, and coordinating infrastructure, software applications and services
  • Orchestration is focused on coordinating automated tasks to work together
  • Automated tasks provide predictable results while executing repetable tasks

Infrastructure Automation

  • Sets up individual tasks to run on their own
  • Builds an asset in the IT supply chain
  • Improves collective efficiencies when combined
  • All industries and sizes look to automation to increase efficiency at scale

Summary

  • Workflow: Includes all sequences like provisioning Cloud resources, deploying application code, and testing the application

  • Orchestrator: Manages the execution of each task, wether manual or automated

  • Tasks: Document how each step corresponds to a specific action, such as launching an EC2 instance, uploading code to a repository, or configuring load balancers.

Additional Definitions

  • Service orchestration is broad approach to provide a complete solution “as-a-service”

  • Cloud orchestration modern equivalent of infrastructure setup by system administrators using tools like Chef, Ansible, Puppet, and others

  • Release orchestration is a combination of deployment automation, pipeline and environment management
